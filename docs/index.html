Welcome to Alexandria, a content management system for librarians and kopimi-ists.

It's open source, too

Introduction

Alexandria is a replacement for the many, many scripts that made up the original PirateBox software. PirateBox was then adapted to handle other systems such as LibraryBox (a more "read only" environment) as well as BiblioBox.

So what does Alexandria try to accomplish? Simple, really: It tries to ecompass all the tasks neccesary for a small, hyperlocal file sharing platform.Alexandria replaces the need for OpenWRT's framework, making the configuration itself agnostic to the underlying Linux distribution underneath.

Alexandria is built on time-tested tech: Python, the Flask framework, and Debian make up the core Alexandria distribution. This means stability in the face of adversity.

Getting Started

When you first start an Alexandria-enabled device, you'll see a wireless network appear local to you. This network will have a name such as "LibraryBox", "PirateBox" or another similar name. Check with your distribution's documentation

The first thing you need to do is change the password that is used to administrate your device. Doing so makes sure that other users aren't going to accidentially change the settings on your device. To do this, you'll want to

  1. Connect to your device's wifi network
  2. Navigate to your device. By default, this will be alexandria.lan -- however, this may change depending on the distribution. Check with your distribution's documentation.
  3. Choose "admin" at the front page
  4. Enter the default password: librarian
  5. Choose "General Configuration" from the list of options
  6. Enter a new password in the "Administrative Password" box
  7. Press "save"

You will be prompted to reboot your device. If you choose not to, all administration pages will inform you that the settings you have changed will not go into effect until you have rebooted your device.

Wireless Network options

To use a self-contained network

This is the default configuration. To change settings related to the network itself (name, security, etc), you'll need to look at hostapd under Service Configuration in administration section. The settings most interesting to people are:

802.11 channel
The channel (1-11) that the network will be on; Commonly, home access points use 3, 6 and 11.
WPA2 security
Turning on this enables encryption on your network using WPA2 security. This is supported by most devices and should be fine for most use. Note that turning this on will require users to know the WPA2 Passphrase.
WPA2 Passphrase
The password used for encryption. If WPA2 security is turned on, all users who wish to connect must know this. It must be between 8 and 63 characters and may contain spaces and special characters (!@#$%, etc.) Using characters from languages other than english is possible, but may not work properly.

Connecting your device to an established network

This scenario involves turning off several services. Notably, you must

  1. Turn off the DNSMasq service
  2. Turn off the HostAPd service
  3. Set the wireless network interface to "client" mode
  4. Enable DHCP or configure IP information.
